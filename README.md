# Standby Task Force (SBTF)

Standby Task Force supports disaster activations using lightweight, high-impact geospatial and crowdsourcing workflows. This repository curates selected public artifacts that demonstrate the breadth of SBTF’s innovation across volunteer coordination, rapid mapping, data pipelines, and actionable intelligence for responders.

## What this repository showcases

The materials in this folder capture how SBTF has combined:

- **Volunteer-powered data collection at scale**
- **Fast, practical geospatial tooling** (often with low-friction platforms)
- **Operational intelligence products** for emergency decision-makers
- **Rapid experimentation and adaptation** during real-world disaster contexts

## Featured documents

### 1) Crowdsourcing Analysis of Imagery
- **File:** `Crowdsourcing Analysis of Imagery.pdf`
- **Focus:** Methods for structuring distributed volunteer analysis of imagery to extract useful disaster intelligence.
- **Innovation highlights:** Human-in-the-loop classification, scalable task decomposition, and rapid signal extraction from visual data.

### 2) MicroMappers Ecuador Simulation (April 2016)
- **File:** `MicroMappers Ecuador simulation April 2016.pdf`
- **Focus:** Simulation-driven validation of digital humanitarian workflows using MicroMappers.
- **Innovation highlights:** Training and stress-testing volunteers, refining categorization schemas, and improving throughput/quality under time pressure.

### 3) SBTF Geo-information for Disasters (QSIC, May 2019)
- **File:** `QSIC - May 2019 - SBTF Geo-information for Disasters.pptx`
- **Focus:** Presentation of SBTF’s geoinformation practices and operational model.
- **Innovation highlights:** End-to-end geospatial activation approaches, collaborative analysis patterns, and practical implementation lessons.

### 4) SBTF-PR-Intel Map Metadata (Public)
- **File:** `SBTF-PR-Intel Map Metadata - Public.pdf`
- **Focus:** Metadata structure and standards for public-facing disaster intel mapping.
- **Innovation highlights:** Transparent data governance, traceability, and map products designed for interoperable operational use.

## Operational approach reflected across these artifacts

Across these documents, recurring SBTF practices include:

- Building workflows that are **simple enough to deploy quickly** but robust enough for crisis operations
- Combining **spreadsheet-native collaboration** with map-based visualization and analysis
- Enabling **quality control and data vetting** in volunteer-generated information streams
- Producing **Who/What/Where-style situational awareness** outputs for responders
- Integrating **social and imagery-based signals** into shared operational pictures

## Why this matters

Disaster response requires tools and processes that can be stood up quickly, understood by diverse volunteers, and trusted by decision-makers. SBTF’s work demonstrates how pragmatic digital workflows can convert distributed volunteer effort into credible, actionable intelligence during high-pressure events.
